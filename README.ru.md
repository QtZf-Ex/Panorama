# Panorama REST API Toolkit

[English version](README.md)

Набор самостоятельных Python-скриптов для повседневного администрирования и security-операций на **Palo Alto Networks Panorama**, построенных напрямую поверх XML/REST API PAN-OS. Без вендорского SDK — только `requests`, `xml.etree` и понимание того, что реально возвращает API.

Каждый скрипт вырос из реальной рабочей задачи (проверять тысячи security-правил руками не масштабируется) и был доведён до отдельного переиспользуемого инструмента.

## Что демонстрирует этот проект

- Работу напрямую с вендорским XML/REST API (аутентификация через `keygen`, чтение/запись конфигурации по XPath, асинхронные log-джобы с поллингом)
- Массовое управление конфигурацией с безопасными дефолтами: любая операция записи сначала печатает план и спрашивает подтверждение `y/N`, изменения попадают только в **Candidate Configuration** — автокоммита нет
- Анализ security-логов: постраничные запросы traffic-логов, группировка по времени, сопоставление src/dst с известными Address Objects
- Общие навыки скриптинга: работа с CSV, параллельное TCP/TLS-сканирование, разбор X.509-сертификатов, CLI, понятный не-разработчику

## Структура репозитория

```
├── address_object_tagging/     Массовое тегирование / переименование / нормализация Address Objects
├── security_rule_logging/      Массовое управление логированием правил
├── security_profile_audit/     Поиск и исправление правил без Security Profile Group
├── url_category_generator/     Генерация wildcard-паттернов для URL Category
├── tls_certificate_scanner/    Сканер портов + TLS-сертификатов с отчётом по срокам действия
├── traffic_log_analyzer/       Универсальный анализатор traffic-логов "кто обращался к X"
└── kerberos_auth_investigator/ Расследование всплесков Kerberos-трафика (порт 88)
```

## Инструменты

| Папка | Скрипт | Что делает |
|---|---|---|
| `address_object_tagging/` | `ADD_TAG.py` | Создаёт/добавляет тег на Address Objects (по имени, значению или списку из CSV); массово переименовывает объекты по паттерну (`{IP}`, `{FQDN}`, `{VALUE}`, `{N}`); массово убирает доменный постфикс из имён объектов. |
| `security_rule_logging/` | `edit_log_start.py` | Проверяет и переключает **Log at Session Start** по выбранным device-group и rulebase. |
| `security_rule_logging/` | `edit_log_forwarding.py` | Массово заменяет профиль **Log Forwarding** на security-правилах — по исходному профилю, по одному правилу или по списку номеров правил из CSV. |
| `security_profile_audit/` | `profile_audit.py` | Находит все security-правила без Security Profile Group (`None` / `Group(None)`), выгружает нумерованный CSV-отчёт, затем применяет исправления из этого же CSV после того, как вы проставите нужный профиль в каждой строке. |
| `url_category_generator/` | `url_category.py` | Превращает плоский список доменов в паттерны `domain/*`, `domain/`, `www.domain`, `www.domain/*`, которые Panorama ожидает для кастомных объектов URL Category. |
| `tls_certificate_scanner/` | `scan_cert.py` | Интерактивный сканер: режим **Поиск** проходит по TCP-портам и снимает цепочки сертификатов через `openssl s_client`; режим **Аудит** проверяет ~80 известных TLS/management-портов (K8s, Elastic, UniFi, admin-порты NGFW/PAM/DLP/NAC и т.д.). Выгружает CSV с полной цепочкой и отдельно leaf-сертификатами, с обратным отсчётом до истечения. Цели можно брать из одного хоста, CSV или тега Panorama. |
| `traffic_log_analyzer/` | `search_bruteforce_mail.py` | Запрашивает traffic-логи Panorama (постраничные log-джобы), считает обращения по src/dst IP, помечает всё, что превышает порог. Есть готовый пресет под SMTP-bruteforce, но подходит и как общий анализатор трафика. |
| `kerberos_auth_investigator/` | `kerberos_investigator.py` | Расследует всплеск Kerberos-трафика (AS-REQ / порт 88) от заданных PAM/jump-host IP: группирует обращения по времени, сопоставляет source ports (детект retry-loop), и вытаскивает контекстный трафик непосредственно перед каждым Kerberos-обращением. Выгружает несколько CSV-разрезов. |

## Установка

```bash
git clone <this repo>
cd <this repo>
pip install -r requirements.txt
```

Для `tls_certificate_scanner/scan_cert.py` также нужен бинарник `openssl` в `PATH` (на Windows скрипт сам предложит поставить его через `winget`, если не найден).

### Настройка доступа к API

В каждой папке инструмента есть свой `api_key.py` — это **шаблон с пустыми значениями**, а не общий секрет:

```python
api_ip = ""     # IP или FQDN management-интерфейса Panorama
api_key = ''    # можно вставить готовый API key сюда, или оставить пустым
user = ""       # логин Panorama (нужен, если api_key пустой)
passwd = ""     # пароль Panorama (нужен, если api_key пустой)
```

`pan_api.py` в каждой папке использует эти значения, чтобы автоматически сгенерировать (через `type=keygen`) и провалидировать API-ключ — достаточно заполнить `api_ip` + `user`/`passwd`, либо `api_ip` + `api_key`.

**Никогда не коммитьте заполненный `api_key.py`.** После клонирования либо редактируйте его локально, не индексируя изменения (`git update-index --skip-worktree <folder>/api_key.py`), либо храните реальные учётные данные в отдельной неотслеживаемой копии. `.gitignore` в этом репозитории уже исключает сгенерированные отчёты, но **не** исключает сам `api_key.py` — пустой шаблон должен оставаться в репозитории.

## Особенности реализации

- Все запросы идут через собственный XML/REST endpoint PAN-OS (`type=config`, `type=log`, `type=op`) с `verify=False` (self-signed сертификат Panorama) — используйте только в доверенном сетевом сегменте.
- Любая операция записи всегда идёт по схеме: **план → подтверждение (`y/N`) → применение**. Изменения попадают в Candidate Configuration, коммит в Panorama вы делаете вручную.
- Интерактивные подсказки и вывод в консоль — на русском (рабочий язык автора); код, структура и README — на английском.
- Пример-CSV в репозитории (`ip_range.csv`, `dns_range.csv` и т.д.) используют документационные адреса (`192.0.2.0/24`, `192.168.0.0/16`) — замените их на свои реальные адреса перед боевым запуском.

## Требования

- Python 3.8+
- `requests` (см. `requirements.txt`)
- `openssl` в `PATH` для `tls_certificate_scanner/`

## Дисклеймер

Проект не аффилирован с Palo Alto Networks и не одобрен ей. Это личные рабочие инструменты, опубликованные как есть — проверяйте их перед запуском на боевой Panorama, особенно всё, что относится к режиму записи. Скрипты проверены на актуальных на момент написания соглашениях XML/REST API PAN-OS — поведение эндпоинтов может отличаться в разных версиях PAN-OS.
